Maltrail is an open-source malicious traffic detection system that passively captures network packets and checks them against curated blacklists and heuristic detection logic. When a threat is detected, events are stored and surfaced through a web-based reporting interface — giving your team clear visibility into what's hitting your network.

## How it works

Maltrail follows a three-component architecture: a **Sensor** captures traffic, a **Server** stores events and serves the web UI, and a **Client** (your browser) renders the reporting dashboard.

<Steps>
  <Step title="Deploy the Sensor">
    Install Maltrail on any Linux machine with network access. The sensor runs with root privileges to capture packets on any interface using `pcapy-ng`.
  </Step>
  <Step title="Start the Server">
    Run `server.py` to start the HTTP reporting server on port 8338. By default, the server co-locates with the sensor and writes logs to `/var/log/maltrail/`.
  </Step>
  <Step title="Access the reporting UI">
    Open `http://<server-ip>:8338` in your browser. Log in and explore the timeline, charts, and event table to investigate detected threats.
  </Step>
  <Step title="Tune and integrate">
    Add custom trail files, configure syslog/Logstash forwarding, and set up fail2ban integration to automate blocking of detected attackers.
  </Step>
</Steps>

## Key capabilities

<CardGroup cols={2}>
  <Card title="100+ threat feeds" icon="shield-halved" href="/configuration/trails">
    Maltrail pulls from feeds like AbuseIPDB, Emerging Threats, Tor exit nodes, URLhaus, and dozens more — automatically updated daily.
  </Card>
  <Card title="Heuristic detection" icon="brain" href="/configuration/sensor">
    Catch unknown threats via port scan detection, excessive NXDOMAIN queries, DGA-like domain names, and direct executable downloads.
  </Card>
  <Card title="Fail2ban integration" icon="ban" href="/operations/fail2ban">
    Expose today's attacker IPs via the `/fail2ban` endpoint and feed them directly into iptables or ipset for automated blocking.
  </Card>
  <Card title="Syslog & Logstash export" icon="arrow-up-right-from-square" href="/operations/integrations">
    Forward events in CEF format to your SIEM or as JSON to Logstash for centralized log management.
  </Card>
</CardGroup>

<Note>
  Maltrail requires Python 2.6, 2.7, or 3.x on a Unix/BSD system, plus the `pcapy-ng` package for packet capture. The sensor component requires root or `CAP_NET_RAW` privileges.
</Note>
