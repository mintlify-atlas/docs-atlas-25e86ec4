---
title: "Get started with Maltrail"
description: "Install Maltrail, start the sensor and server, verify your first detection against a known-malicious IP, and access the web reporting interface."
---

This guide walks you through installing Maltrail, starting the sensor and server processes, confirming that detection is working, and opening the reporting interface. The default configuration monitors all interfaces and writes logs to `/var/log/maltrail/`.

<Note>
  Maltrail depends on `pcapy-ng` for packet capture — not the older `pcapy` library. Always install `pcapy-ng` via pip. The legacy package is deprecated and broken under Python 3.
</Note>

<Steps>
  <Step title="Install dependencies">
    Install system packages and the `pcapy-ng` Python library for your distribution.

    <CodeGroup>

    ```bash Ubuntu/Debian
    sudo apt-get install git python3 python3-dev python3-pip python-is-python3 libpcap-dev build-essential procps schedtool
    sudo pip3 install pcapy-ng
    ```

    ```bash SUSE/openSUSE
    sudo zypper install gcc gcc-c++ git libpcap-devel python3-devel python3-pip procps schedtool
    sudo pip3 install pcapy-ng
    ```

    </CodeGroup>
  </Step>

  <Step title="Clone the repository">
    Clone Maltrail with a shallow fetch to keep the download small.

    ```bash
    git clone --depth 1 https://github.com/stamparm/maltrail.git
    cd maltrail
    ```
  </Step>

  <Step title="Enable promiscuous mode (optional)">
    For monitoring on a SPAN or mirror port, place your interfaces into promiscuous mode so the sensor receives all frames on the segment — not just those addressed to the host.

    <Tip>
      This step is recommended for production deployments on a dedicated monitoring interface. You can skip it for a basic local test where the sensor captures only traffic to and from the host itself.
    </Tip>

    ```bash
    for dev in $(ifconfig | grep mtu | grep -Eo '^\w+'); do ifconfig $dev promisc; done
    ```
  </Step>

  <Step title="Start the sensor">
    Run the sensor with root privileges. On first run (or after a long period offline), Maltrail automatically downloads trail definitions from its feeds before starting to monitor traffic.

    ```bash
    sudo python3 sensor.py
    ```

    The sensor begins monitoring the interface specified by `MONITOR_INTERFACE` in `maltrail.conf` — the default is `any`, which captures from all available interfaces.
  </Step>

  <Step title="Start the server">
    Open a new terminal, navigate to the `maltrail` directory, and start the server. The server does not need root privileges.

    ```bash
    python3 server.py
    ```

    The server listens on port `8338` for the HTTP reporting interface and, when `UDP_ADDRESS` is configured, on port `8337` for incoming events from remote sensors.
  </Step>

  <Step title="Verify detection">
    Test that the sensor is correctly detecting traffic by pinging a known-malicious IP address included in Maltrail's static trails, then inspecting today's log file.

    ```bash
    ping -c 1 136.161.101.53
    cat /var/log/maltrail/$(date +"%Y-%m-%d").log
    ```

    You should see a CSV log entry for the ping event. You can also test DNS trail detection:

    ```bash
    nslookup morphed.ru
    cat /var/log/maltrail/$(date +"%Y-%m-%d").log
    ```
  </Step>

  <Step title="Access the reporting interface">
    Open your browser and go to `http://127.0.0.1:8338`. Log in with the default credentials:

    - **Username:** `admin`
    - **Password:** `changeme!`

    The reporting interface displays a sliding timeline, summary charts (events, sources, threats, trails), and a paginated table of threat entries for the selected 24-hour period.

    <Warning>
      Change the default password before exposing the server on any network. User credentials are configured in `maltrail.conf` under the `USERS` section using the format `username:sha256(password):UID:filter_netmask`.
    </Warning>
  </Step>
</Steps>

## Docker alternative

If you prefer a containerised setup, a pre-built server image is available. Note that only the server component is packaged in the container — the sensor must still run on the host.

```bash
docker run -d --name maltrail --restart=unless-stopped \
  -p 8338:8338/tcp \
  -p 8337:8337/udp \
  -v /etc/maltrail.conf:/opt/maltrail/maltrail.conf:ro \
  ghcr.io/stamparm/maltrail:latest
```

## Stopping Maltrail

To stop the sensor and server when running in the background:

```bash
sudo pkill -f sensor.py
pkill -f server.py
```
