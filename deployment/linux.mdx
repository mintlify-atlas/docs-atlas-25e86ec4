---
title: "Deploy Maltrail on Linux (production setup)"
description: "Install and configure Maltrail on Linux for production use, with cron-based autostart, log directories, and privilege setup for the sensor."
---

Following these steps gives you a production-grade Maltrail installation at `/opt/maltrail` with configuration at `/etc/maltrail/maltrail.conf`, log output at `/var/log/maltrail`, and cron jobs that keep both the server and sensor running automatically across reboots and daily trail updates.

<Steps>
  <Step title="Install dependencies and clone the repository">
    Install system packages, the `pcapy-ng` Python library, and clone Maltrail into `/opt/maltrail`.

    <CodeGroup>

    ```bash Ubuntu/Debian
    sudo apt-get install git python3 python3-dev python3-pip python-is-python3 libpcap-dev build-essential procps schedtool
    sudo pip3 install pcapy-ng
    cd /tmp
    git clone --depth 1 https://github.com/stamparm/maltrail.git
    sudo mv /tmp/maltrail /opt
    sudo chown -R $USER:$USER /opt/maltrail
    ```

    ```bash SUSE/openSUSE
    sudo zypper install gcc gcc-c++ git libpcap-devel python3-devel python3-pip procps schedtool
    sudo pip3 install pcapy-ng
    cd /tmp
    git clone --depth 1 https://github.com/stamparm/maltrail.git
    sudo mv /tmp/maltrail /opt
    sudo chown -R $USER:$USER /opt/maltrail
    ```

    </CodeGroup>

    <Note>
      Use `pcapy-ng`. The older `pcapy` package is deprecated and causes issues in Python 3 environments.
    </Note>
  </Step>

  <Step title="Set up the working environment">
    Create the log and configuration directories, copy the default configuration, and edit it to match your environment.

    ```bash
    sudo mkdir -p /var/log/maltrail
    sudo mkdir -p /etc/maltrail
    sudo cp /opt/maltrail/maltrail.conf /etc/maltrail
    sudo nano /etc/maltrail/maltrail.conf
    ```
  </Step>

  <Step title="Configure cron jobs for autostart and updates">
    Add cron entries to keep the server and sensor running and to pull trail updates daily.

    Run `crontab -e` (as your regular user) and add the following lines for the **server**:

    ```
    */5 * * * * if [ -n "$(ps -ef | grep -v grep | grep 'server.py')" ]; then : ; else python3 /opt/maltrail/server.py -c /etc/maltrail/maltrail.conf; fi
    0 1 * * * cd /opt/maltrail && git pull
    ```

    Run `sudo crontab -e` and add the following lines for the **sensor**:

    ```
    */1 * * * * if [ -n "$(ps -ef | grep -v grep | grep 'sensor.py')" ]; then : ; else python3 /opt/maltrail/sensor.py -c /etc/maltrail/maltrail.conf; fi
    2 1 * * * /usr/bin/pkill -f maltrail
    ```

    The server cron checks every 5 minutes and pulls updates at 01:00. The sensor cron checks every minute and performs a nightly restart at 01:02 to pick up trail updates.
  </Step>
</Steps>

<Note>
  The sensor component requires at least 1 GB of RAM when running in single-process mode. If `PROCESS_COUNT` is set above `1` in `maltrail.conf`, memory usage increases depending on the value set for `CAPTURE_BUFFER`.
</Note>

<Note>
  The sensor requires administrative privileges to capture raw packets. You can either run it as `root` or grant the specific capabilities to the Python binary:

  ```bash
  sudo setcap 'CAP_NET_RAW+eip CAP_NET_ADMIN+eip' /bin/python3
  ```
</Note>

<Tip>
  Put your monitoring interfaces into promiscuous mode so the sensor captures all traffic passing through the interface, not just traffic addressed to the host:

  ```bash
  for dev in $(ifconfig | grep mtu | grep -Eo '^\w+'); do ifconfig $dev promisc; done
  ```
</Tip>
